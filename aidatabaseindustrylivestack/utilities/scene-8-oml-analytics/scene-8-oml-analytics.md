# Scene 8 OML Analytics

## Introduction

This scene shows in-database machine learning for demand surge scoring, customer segmentation, operational value forecasting, vector clusters, and capacity intelligence.

Estimated Time: 8 minutes

![OML Analytics page showing ML scorecards and tabs.](images/scene-08-oml-analytics.png)

### Objectives

In this lab, you will:
- Open the OML analytics workflow.
- Use the tabs to compare ML use cases.
- Refresh a model view and explain the model signal in operator terms.

## Task 1: Review analytics scorecards

1. Click **OML Analytics** in the sidebar.
2. Review the top scorecards for demand surge, segmented customers, operational value model R2, and active ML models.
3. Select each analytics tab to show the different model views.

Expected result:
- The page frames OML as operational decision support, not a separate data-science notebook.
- Each tab keeps model output near the business context it supports.
## Task 2: Refresh an analytics view

1. Select a time window or forecast horizon where the page provides controls.
2. Click **Refresh**.
3. Compare the chart, list, or cluster output with the right-side Oracle evidence.

Expected result:
- With the full stack running, the page scores or forecasts directly against Oracle data.
- The evidence panel connects the workflow to DBMS_DATA_MINING, prediction functions, clustering, regression, and SQL-side scoring.

## Task 3: Why this matters?

The OML scene gives utilities leaders predictive context without moving operational data out of the database. That matters when speed, governance, and repeatability all matter.
