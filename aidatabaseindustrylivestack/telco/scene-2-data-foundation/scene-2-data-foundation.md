# Scene 2 Data Foundation

## Introduction

This scene prepares the Seer Comms dataset that powers the rest of the LiveStack Demo. The page loads or restores the governed Oracle AI Database 26ai data foundation, then shows the main data domains and data types used across the South Florida 5G demand-surge story and the eight active telecom use cases.

Estimated Time: 5 minutes

![Data Foundation page with dataset status and loaded telecom data domains](images/data-foundation.png)

### Objectives

In this scene, you will:
- Load the Seer Comms demo dataset, or restore it if data is already loaded.
- Understand what data is prepared for the LiveStack Demo.
- Review the **What Gets Loaded** carousel to see the data domains and data types used across the demo.
- Continue to the next scene after the dataset is ready.

## Task 1: Prepare the dataset

![Prepare the Dataset action area](images/prepare-dataset-action.png)

1. Click **Data Foundation** in the sidebar.
2. In **Prepare the Dataset**, click **Load Demo Data**.
3. If the dataset is already loaded, click **Restore Demo Data** to reset the demo data to a clean state.
4. Wait for the progress indicator to finish and review the refreshed record counts.

The load prepares telecom services, subscriber signals, service orders, network sites, demand regions, field dispatch context, vector embeddings, graph relationships, machine learning outputs, and agent audit history. In the current demo dataset, the data foundation includes **32** telecom services, **5,000** subscriber signals, **3,000** service orders, **12** network sites, **360** demand forecasts, **5,000** signal vectors, **1,572** semantic matches, and **36** telecom impact entities.

## Task 2: Review what gets loaded

![Loaded telecom data domains and record counts](images/loaded-data-domains.png)

1. Scroll to **What Gets Loaded**.
2. Read the three visible carousel tiles.
3. Use the right carousel arrow to review the remaining tiles.
4. Look for the major data types used in the demo: relational tables, JSON duality views, graph relationships, spatial geometry, vector embeddings, machine learning results, and agent audit records.

You can move to the next scene.
