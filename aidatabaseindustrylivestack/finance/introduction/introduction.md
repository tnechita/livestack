# Seer Equity Bank Finance LiveStack

## Introduction

This workshop is the scene-by-scene runbook for the Seer Equity Bank Finance LiveStack. The application shows how Oracle AI Database 26ai can serve trusted finance data across dashboards, regulatory signal search, financial crime graph analysis, spatial service coverage, transaction documents, in-database machine learning, natural-language SQL, and AI agent workflows.

Estimated Demo Time: 90 minutes

### Objectives

In this workshop, you will:
- Navigate the finance LiveStack using the left application menu and top-level operator controls.
- Explain how Seer Equity Bank uses one Oracle data foundation for risk, service, revenue, and compliance workflows.
- Inspect converged data patterns: relational tables, JSON duality views, property graph, spatial geometry, vectors, Oracle Machine Learning, Select AI style query execution, and agent audit trails.
- Run or describe the portable Podman Compose stack using the included `livestack-finance.zip` package.

### Prerequisites

This workshop assumes you have:
- Access to the running LiveStack application.
- A browser session open to the Seer Equity Bank Finance LiveStack.
- Basic familiarity with financial services operations, fraud investigation, regulatory signals, and client service routing.
- For the download lab, Podman, Podman Compose, and enough local resources to run Oracle Database Free, ORDS, Ollama, and the app container.

## Workshop Flow

1. Download and run the LiveStack.
2. Scene 1: Review the Seer 26ai data foundation.
3. Scene 2: Read the risk and operations dashboard.
4. Scene 3: Search regulatory and market signals with vectors.
5. Scene 4: Investigate financial crime networks with graph traversal.
6. Scene 5: Inspect client service coverage with Oracle Spatial.
7. Scene 6: Compare transaction rows with JSON duality documents.
8. Scene 7: Review predictive risk and revenue analytics.
9. Scene 8: Ask plain-English questions over the governed schema.
10. Scene 9: Route questions through the agent console.
11. Scene 10: Use the dataset manager to validate or restore finance data.
12. Conclusion and business outcomes.
