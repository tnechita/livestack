# Scene 1: Seer Transport Control Tower

## Introduction

This scene frames the LiveStack as a transportation control tower. The user sees the core workflow stages and chooses a starting path into fleet operations, shipper signals, or natural language data exploration.

Estimated Time: 8 minutes

![Seer Transport control tower welcome screen](images/control-tower-welcome.png)

### Objectives

In this lab, you will:
- Orient the user to the left navigation rail and control tower story.
- Use the welcome quick actions to enter the demo through different operational roles.
- Connect the opening screen to Oracle AI Database 26ai as the shared data and AI platform.

## Task 1: Open the control tower

1. Open the application home page.
2. Review the left navigation rail and confirm the first scene is **Seer Transport Control Tower**.
3. Inspect the workflow stages: Ingest, Transform, Serve Data, Serve AI, and Govern.
4. Review the capability cards for converged queries, natural language to SQL, graph and spatial analytics, vector search, and REST access to data.

Expected result:
- The user understands that the LiveStack is not a single dashboard. It is a guided transportation operations workflow powered by one Oracle data platform.

## Task 2: Use a quick action

1. Click **Open fleet command center**.
2. Return to the welcome scene using the navigation rail.
3. Click **Review shipper signals**.
4. Return again and click **Ask Seer Transport data**.

Expected result:
- Each quick action moves to a real downstream scene.
- The user sees that the control tower is an entry point into operations, signal monitoring, and natural language analysis.

## Task 3: Why this matters?

Transportation operations teams do not all start from the same question. A dispatcher may start with capacity, a customer team may start with a shipper signal, and an executive may ask a natural language question. The control tower makes those paths feel like one connected workflow instead of separate tools.
