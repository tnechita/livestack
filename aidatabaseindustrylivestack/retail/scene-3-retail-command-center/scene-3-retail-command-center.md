# Scene 3 Retail Command Center

## Introduction

The command center is the executive and operator overview. It pulls together return exposure, demand signals, product velocity, inventory risk, social activity, and agent status so the presenter can show the retail problem before drilling into individual workflows.

Estimated Time: 10 minutes

![Retail command center dashboard](images/retail-command-center.png)

### Objectives

In this lab, you will:
- Open **Retail Command Center**.
- Review the dashboard indicators and product signal table.
- Use search, filter, or refresh controls to demonstrate an operator workflow.

## Task 1: Open the command center

1. Click **Retail Command Center** in the sidebar.
2. Review the KPI cards and trend sections.
3. Inspect the product table and any visible Oracle evidence panel.

Expected result:
- The dashboard shows the current retail operating picture.
- The audience sees return pressure, demand movement, and inventory context in one page.

## Task 2: Explore a product signal

1. Use the product search field to search for a brand, product, or return signal.
2. Click a product row if product details are visible.
3. Review the resulting detail panel, JSON payload, or SQL-backed evidence.

Expected result:
- The page narrows the dashboard context to the selected signal.
- The presenter can move from summary metrics to product-level evidence without leaving the app.

## Task 3: Why this matters?

Retail operations need a single place to see where revenue, returns, demand, and inventory risk are converging. The command center turns the underlying data foundation into an operational first stop for executives and line-of-business users.
