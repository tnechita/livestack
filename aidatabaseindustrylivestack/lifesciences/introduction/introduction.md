# Seer Lifesciences LiveStack Demo Guide

## Introduction

This workshop is the runbook for the Seer Lifesciences LiveStack. It follows the running application scene by scene so a presenter can move from a regulated operations story to the exact screens, controls, expected state changes, and business signals that the audience should remember.

Estimated Demo Time: 1 hour 45 minutes

![Seer Lifesciences control tower screenshot](../scene-1-seer-lifesciences-control-tower/images/control-tower.png)

### Objectives

In this workshop, you will:
- Walk through the visible Seer Lifesciences application scenes in the same order as the app navigation.
- Connect each workflow to the Oracle AI Database 26ai feature it demonstrates.
- Use real app controls such as filters, graph depth, map layers, JSON duality tabs, Ask Data modes, and agent prompts.
- Explain the business signal behind each scene for regulated supply, quality, cold-chain, and AI-assisted operations.
- Download and run the portable LiveStack package with Podman Compose.

### Prerequisites

This workshop assumes you have:
- Access to the Seer Lifesciences LiveStack application.
- A browser open to the application URL.
- The `livestack-lifesciences.zip` package if you plan to run the stack locally.
- Podman or a compatible Compose runtime for the download lab.
- Enough local resources to run Oracle Database Free, ORDS, Ollama, and the app container when using the portable stack.

## Workshop Flow

- Scene 1: Seer Lifesciences Control Tower.
- Scene 2: Seer 26ai Data Foundation.
- Scene 3: Quality and Supply Operations Dashboard.
- Scene 4: Regulatory and Quality Signals.
- Scene 5: Quality Propagation Network.
- Scene 6: Cold-Chain Service Coverage.
- Scene 7: Clinical Supply Orders and Deviations.
- Scene 8: Predictive Quality and Supply Analytics.
- Scene 9: Ask Seer Lifesciences Data.
- Scene 10: Agent Console.
- Conclusion and business outcomes.
- Download the LiveStack and run the portable stack with Podman Compose.
