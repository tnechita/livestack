# High Tech Product Intelligence LiveStack Guide

## Introduction

This guide is the scene-by-scene runbook for the High Tech Product Intelligence LiveStack. The demo follows Seer Tech as it turns product portfolios, enterprise buyer signals, developer ecosystem relationships, solution orders, spatial availability, machine learning scores, natural-language questions, and AI agent actions into one governed product-intelligence workflow.

Estimated Demo Time: 90 minutes

![Seer Tech Control Tower rendered in the LiveStack app](../scene-1-seer-tech-control-tower/images/scene-1-seer-tech-control-tower.png)

### Objectives

In this workshop, you will:
- Run the LiveStack locally or in a hosted workshop environment.
- Follow each visible application scene in the same order as the left navigation.
- Connect every operator interaction to the Oracle AI Database 26ai capability behind it.
- Use the final conclusion to turn the demo into a customer-facing product-intelligence narrative.

### Prerequisites

- A workstation, sandbox, or cloud tenancy with Podman or a compatible Compose runtime.
- Network access for Oracle Database Free, ORDS, Ollama, and the ONNX model download used by the stack.
- The archive livestack-hightech.zip, extracted to a folder named hightech.
- Enough local resources for Oracle Database Free, ORDS, Ollama, and the Node application containers.

## Workshop Flow

- Scene 1: Seer Tech Control Tower - This opening scene frames the Seer Tech Product Intelligence LiveStack as one connected workflow for product signals, capacity risk, governed data products, and AI-assisted operations.
- Scene 2: Seer Tech 26ai Data Foundation - This scene explains the data model behind the app: relational product operations, JSON signals, graph relationships, spatial availability, vector search, machine learning, and agent audit trails in one Oracle Database-centered architecture.
- Scene 3: Product Intelligence Command Center - The command center gives product and operations leaders a first read on solution orders, product value, high-urgency buyer signals, high-demand products, and recent agent activity.
- Scene 4: Enterprise Buyer Signal Monitor - This scene demonstrates signal monitoring over synthetic buyer and developer activity, including vector search across product embeddings and signal embeddings.
- Scene 5: Developer Ecosystem Graph - The graph scene visualizes developer advocates, influencer connections, brand links, and product mentions so the operator can understand how market signals propagate through the developer ecosystem.
- Scene 6: Product Availability and Capacity Map - This scene demonstrates spatial product availability with fulfillment sites, routes, service zones, demand overlays, buyer risk tiers, and VPD-aware access.
- Scene 7: Solution Orders - The orders scene shows solution orders as operational rows, JSON relational duality documents, and fulfillment route evidence, all filtered through governed application access.
- Scene 8: OML Product Intelligence - The OML scene demonstrates in-database machine learning for demand surge prediction, enterprise buyer segmentation, value forecasting, vector clustering, and capacity intelligence.
- Scene 9: Ask Seer Tech Data - Ask Seer Tech Data gives users a natural-language interface over the live schema, with modes for narration, chat, SQL inspection, and SQL execution.
- Scene 10: AI Agent Console - The AI Agent Console demonstrates application-layer agents that use Oracle SQL, PL/SQL tools, vector retrieval, in-database scoring, and JSON audit events to support product operations.
